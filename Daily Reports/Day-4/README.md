# Day 4 Report: ESP32 and ThingSpeak Integration

## Objective

The goal of this project is to learn about cloud based internet of things platforms and figure out how ESP32 devices can send data to ThingSpeak so we can monitor and visualize it from anywhere.

## Activities Performed

- I made a ThingSpeak account. Created a channel for this project.

- I set up fields to track the health of a drone.

- I got the Write API Key from ThingSpeak.

- I wrote a program in MicroPython using Wokwi to connect the ESP32 to Wi-Fi.

- I used the ESP32 to send the battery level to ThingSpeak.

- I looked at the graphs on the ThingSpeak dashboard to see what the data looks like.

## Learning Outcomes

- I learned what internet of things cloud platforms are and how they work.

- I found out what channels and fields and API keys are used for in ThingSpeak.

- I learned how to connect the ESP32 to the internet using Wi-Fi.

- I saw how to send data from the ESP32 to the cloud.

- I got to see data in time on the ThingSpeak graphs.

## Conclusion

Successfully connected the ESP32 to the ThingSpeak cloud platform and sent it to the battery level data. This project showed me the steps for monitoring the health of a drone using the cloud and visualizing internet of things data. The ESP32 and ThingSpeak integration worked well. Overall I learnt about the ESP32 and ThingSpeak.
